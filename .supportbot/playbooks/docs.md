# Documentation Issues Playbook

## Types of Documentation Issues

### Typos and Grammar
- Simple fixes, usually safe to accept PR immediately
- Check for consistency with existing documentation style

### Missing Information
- Identify which sections need expansion
- Consider target audience (beginner vs. advanced)
- Check if information exists elsewhere

### Incorrect Examples
- Verify code examples actually work
- Test commands before documenting them
- Ensure examples use current API

### Outdated Content
- Check if refers to deprecated features
- Update version numbers and compatibility info
- Link to migration guides if applicable

## Diagnostic Steps
1. Locate the specific document/page
2. Verify the issue exists in latest version
3. Check if similar docs have correct information
4. Test any code examples

## Safe Next Steps
- For typos: Quick fix with PR review
- For missing info: Outline sections needed
- For examples: Test and update code
- For outdated: Check changelog for correct info
